Небольшая, композируемая коллекция агентских навыков (skills) для кода,
исследований и оркестрации нескольких агентов. Каждый навык самодостаточен,
основан на доказательствах и поставляется с эталонной документацией с
прогрессивным раскрытием деталей.

> Создано [anshmajumdar](https://github.com/anshmajumdar121). Навыки
> доработаны на основе открытых исследований, плана исполнения
> GPT-5.6 Sol, а также коллекции
> [BuilderIO/skills](https://github.com/BuilderIO/skills).

## Быстрый старт

```bash
# Клонируем репозиторий
git clone https://github.com/anshmajumdar121/skill-x.git
cd skill-x

# Выбираем навык для установки (ручное копирование)
cp -r skills/think-like-gpt-5-6   ~/.mavis/agents/<your-agent>/skills/
cp -r skills/efficient-fable      ~/.mavis/agents/<your-agent>/skills/
cp -r skills/skill-zero           ~/.mavis/agents/<your-agent>/skills/
cp -r skills/super-skill          ~/.mavis/agents/<your-agent>/skills/

# Или устанавливаем все сразу
for d in skills/*/; do
  cp -r "$d" ~/.mavis/agents/<your-agent>/skills/
done

# Проверяем, что они зарегистрировались
mavis skill list
```

> Замените `<your-agent>` на имя вашего агента (например, `mavis`,
> `claude`, `codex` — там, где хранятся ваши навыки).

### Какой навык когда использовать

- **У меня нетривиальная задача, и мне нужны проверяемые артефакты.**
  → `think-like-gpt-5-6`
- **У меня работа, насыщенная кодовой базой, которую можно
  распараллелить, и я хочу сэкономить токены.** → `efficient-fable`
- **Я хочу отслеживать / направлять внутреннее состояние кодирующего
  агента (механистическая интерпретируемость).** → `skill-zero`
- **Мне нужны и строгость, И эффективность делегирования — в одном
  навыке.** → `super-skill`
- **Мне нужен навык, который со временем сам себя улучшает, с жёсткими
  защитными механизмами против мусора.** → `super-skill` (единственный
  с протоколом самоэволюции)

## Подробности по навыкам

### `/think-like-gpt-5-6`

Применяет фреймворк исполнительного интеллекта GPT-5.6 Sol. Превращает
несовершенный запрос в проверенный, документируемый результат через
**цикл из 7 шагов**: Наблюдение → Интерпретация → Решение → Действие →
Проверка → Исправление → Запись. Опирается на **10 руководящих
принципов**, **архитектуру из 9 этапов**, **реестр рисков из 20
строк**, **6 критериев приёмки + 10 критериев отклонения**,
**состязательное ревью из 18 вопросов** и **чек-лист из 7 разделов**
перед сдачей.

Лучше всего подходит для: многошагового написания кода, исследований
с цитированием источников, создания артефактов, операционных действий,
рекомендаций с высокими ставками и любых задач, где вы хотите
проверяемое рассуждение, а не гладкий ответ «чёрного ящика».

Подробнее: [skills/think-like-gpt-5-6/SKILL.md](../skills/think-like-gpt-5-6/SKILL.md)

### `/efficient-fable`

Использует дорогую модель топового уровня в роли **оркестратора,
архитектора, синтезатора и финального судьи**. Более дешёвые субагенты
выполняют **ограниченную по объёму тяжёлую работу** (сканирование
больших репозиториев, свёртку длинных логов, точечные патчи кода,
проверку через браузер/тесты). **Протокол проверки** гласит:
*относитесь к отчётам субагентов как к наводкам, а не фактам* — всегда
открывайте заново указанный файл, подтверждайте ссылку на строку и
просматривайте diff перед сдачей.

**Паттерн делегирования из 5 шагов**: назвать дорогостоящий риск →
разбить на параллельные части → использовать дешёвых агентов для
тяжёлой работы → запросить сжатые доказательства → тратить токены
оркестратора на уровень принятия решений.

Лучше всего подходит для: работы с насыщенной кодовой базой, которую
можно разбить на независимые параллельные части. Не подходит для
тривиальных задач, работы с высокими ставками из единственного
источника или когда дешёвые субагенты недоступны.

Подробнее: [skills/efficient-fable/SKILL.md](../skills/efficient-fable/SKILL.md)

### `/skill-zero`

Специалист по **механистической интерпретируемости LLM кодирующих
агентов** через линейные зонды на остаточном потоке. Основан на работе
Silva, Tu и Monperrus 2026
([arXiv:2607.05188](https://arxiv.org/abs/2607.05188)).

Навык учит вас:

- **Предсказывать результаты правок** до того, как они попадут на
  диск. Обучить логистическую регрессию (зонд) на скрытых состояниях,
  чтобы считывать, приведёт ли готовящаяся правка к регрессии
  (свойство ◆ Regression) или к провалу тестов (● Full Correctness).
- **Понимать, продвигается ли агент** в середине траектории. Зонд
  ■ Partial Correctness считывает уверенность агента в направлении
  прохождения тестов по его скрытым состояниям (AUC ~0.84 в статье).
- **Заглядывать на ~25 шагов вперёд в план агента.** Открытие
  «латентного горизонта программирования» (latent programming
  horizon): зонд может заранее, за несколько шагов, сигнализировать,
  что траектория движется к провальному состоянию.

Навык включает конкретные цифры из статьи (AUC 0.83, горизонт k≈25,
паттерн перевёрнутой U на средних слоях), 4 канонических свойства,
дисциплину контрольной группы с перемешанными метками (shuffled-label
control), детали переноса между бенчмарками и 5 конкретных режимов
отказа.

Подробнее: [skills/skill-zero/SKILL.md](../skills/skill-zero/SKILL.md)

### `/super-skill`

Синтез: **строгость + делегирование** в одном навыке. Два режима
работы:

- **Режим строгости** (по умолчанию) — использует цикл из 7 шагов
  GPT-5.6 Sol, 10 принципов, архитектуру из 9 этапов, классификацию
  задач, реестр рисков, уровни валидации, критерии приёмки/отклонения,
  состязательное ревью и чек-лист перед сдачей.
- **Режим делегирования** — для работы, насыщенной кодовой базой или
  токенами, которую можно разбить на независимые параллельные части;
  использует паттерн делегирования Efficient Fable с протоколом
  проверки.

Режим выбирается на этапе приёма задачи. Навык также включает
**5 рисков, специфичных для делегирования** (R-D1–R-D5), 9-й тест
(**T-09: проверка отчётов субагентов**) и протокол проверки,
встроенный в матрицу тестов.

**Саморазвивающийся.** Навык может расти, улучшаться и подрезать сам
себя со временем, под строгой внутренней **конституцией из 7 пунктов**,
которая не даёт появляться мусору. Конституция (в
`skills/super-skill/references/self-evolution.md`):

- **C-1** На основе доказательств — каждое изменение должно ссылаться
  на статью, официальную документацию, код или проверенное наблюдение
- **C-2** Ограниченная область — только фреймворки строгости и
  делегирования, не пользовательские проекты
- **C-3** Консервативность — никаких спекулятивных добавлений
- **C-4** Минимальная планка качества — новый контент должен быть не
  хуже того, что он заменяет
- **C-5** Обратимость — у каждого изменения есть строка в журнале
  изменений
- **C-6** Фильтр против мусора — отклонять расплывчатое наполнение
- **C-7** Приватность — никаких персональных данных, никакого
  контента, специфичного для конкретного проекта

Провал любого отдельного пункта = отклонение изменения. Без
исключений.

Подробнее: [skills/super-skill/SKILL.md](../skills/super-skill/SKILL.md)

## Установка

Навыки спроектированы так, чтобы их можно было просто скопировать в
папку навыков любого агента. Есть три пути установки — выбирайте тот,
что поддерживает ваш агент.

### Путь A — Установка через плагин / маркетплейс (рекомендуется)

Этот репозиторий поставляется как **маркетплейс плагинов Claude Code**
и **плагин Codex**, поэтому вы можете установить его одной командой:

```bash
# Claude Code
/plugin marketplace add anshmajumdar121/skill-x
/plugin install skill-x@skill-x

# Codex
codex plugin install anshmajumdar121/skill-x

# Или обычный npx (Vercel skills CLI)
npx skills@latest add anshmajumdar121/skill-x --skill super-skill
```

Полную матрицу установки по агентам (Claude Code, Codex, OpenCode,
Copilot, а также резервный вариант `cp -r`) смотрите в
[PLUGIN_INSTALL.md](../PLUGIN_INSTALL.md).

### Путь B — Ручная установка (работает для любого агента)

```bash
# Клонируем
git clone https://github.com/anshmajumdar121/skill-x.git
cd skill-x

# Устанавливаем один навык
cp -r skills/super-skill ~/.mavis/agents/<your-agent>/skills/

# Устанавливаем все
for d in skills/*/; do
  cp -r "$d" ~/.mavis/agents/<your-agent>/skills/
done

# Проверяем
mavis skill list
```

### Пути установки по агентам

| Агент | Путь к навыкам | Плагин? |
|---|---|---|
| mavis | `~/.mavis/agents/<name>/skills/` | нет |
| Claude Code | `~/.claude/skills/` (пользователь) или `.claude/skills/` (проект) | да — `.claude-plugin/marketplace.json` |
| Codex CLI | `~/.codex/skills/` (пользователь) или `.codex/skills/` (проект) | да — `.codex-plugin/plugin.json` |
| OpenCode | `~/.config/opencode/skills/` (пользователь) или `.opencode/skills/` (проект) | только вручную |
| GitHub Copilot | `.github/skills/` (проект) или пользовательская папка VS Code | только вручную |

### Проверка корректности установки навыка

```bash
# Проверить линтером один навык
node $(mavis skill show skill-creator | python3 -c 'import json,sys;print(json.load(sys.stdin)["location"])' | xargs dirname)/scripts/lint-skill.js skills/super-skill/

# Проверить линтером все навыки в этом репозитории
./scripts/lint-all.sh
```

## Самоэволюция (только для `/super-skill`)

<br>

![Самоэволюция: обнаружение, доказательства, проверка по конституции, применение, верификация](../media/banner.png)

<br>

`super-skill` — единственный навык, который развивает сам себя.
Протокол описан в
[`skills/super-skill/references/self-evolution.md`](../skills/super-skill/references/self-evolution.md).
Триггеры:

- **Явный**: «evolve this skill», «superskill evolve»
- **Обнаружение устаревшего контента**: конкретное число, ссылка или
  утверждение явно устарели
- **Обнаружение лучшей техники**: становится доступна явно более
  удачная техника
- **Обнаружение мусорного контента**: раздел больше не оправдывает
  своё место

Процедура из 5 шагов: Обнаружение → Доказательства → Проверка по
конституции (все 7 пунктов) → Применение → Верификация (повторный
линт). Журнал изменений в начале `super-skill/SKILL.md` фиксирует
каждое изменение.

## Участие в разработке

Мы приветствуем вклад со стороны сообщества. Порядок работы описан в
[CONTRIBUTING.md](../CONTRIBUTING.md), а рекомендации по планке качества —
во встроенном **протоколе самоэволюции**. Кратко:

1. Каждый навык самодостаточен. Сохраняйте это свойство.
2. Справочная документация построена по принципу прогрессивного
   раскрытия — `SKILL.md` это titульная часть, `references/*.md` —
   детали.
3. Обязательно прохождение линта: `./scripts/lint-all.sh`.
4. Никаких внешних сетевых вызовов во время выполнения. Навыки — это
   статический markdown.
5. Никаких персональных данных, никакого контента, специфичного для
   конкретного проекта.
